Arrays

Everything important about arrays, covered here.

Array Code:

Code for creating and manipulating arrays:

String [] array = new String [10]; // creating a String array of size 10
array[0] = "First"; // setting the first entry to "First"
array[9] = "Last"; // setting the last entry to "Last"
array[4] = array[0]+array[9] // setting a middle entry to "FirstLast"
int [][] array2D = new int [4][4] // creating a 4*4 array of integers
array2D[1] = new int [5]; // now the array row lengths are: 4, 5, 4, 4
array2D[1][4] = 16; // setting a value in the 2D array to 16

Arrays are a very useful tool in programming. They allow you to store multiple, related values, which is helpful, especially when the amount of these related values can vary. They also allow you to perform operations on these values quickly. To declare one, use the following syntax:

<data type> [] arrayName = new <same data type> [<size of array>];

You can replace <data type> (without the <>, of course) with any data type such as int, double, String, a class, or even another array (which we will discuss later in this section). Notice that you have to put the square brackets [] in the declaration, as this specifies that the variable will be an array as opposed to a variable. The square brackets are also used to initialize the array. Inside these brackets you should specify the size of the array, which should be an integer. 

 Depending on the type, the values of the array, once initialized should have default values. For numbers, like int or double, the value is 0 (or 0.0). For Strings, the default value is a blank String (“”). For anything else, it’s null. Be careful with default values, especially the null ones!

At this point, we can treat each individual element in the array as its own variable. To access an element, we need to specify the name of the array as well as the location of the element: arrayName[<index>]. Note that indices start from 0, so if we want to access the third element in the array, we need to type 2 in place of <index>. For example, if we had an array like this:

If we want to access the third element, we would need to type arrayName[2]. Notice how the indices start from 0 and not 1. If we wanted the indices to start at 1, we would have to add a filler value at the beginning. For example, if we wanted the array to be one based (indices start at 1) as opposed to being zero based (indices start at 0), we would have to initialize the array with one more element than the actual length of the array. However, if we already initialized the array to be zero based, then we would have to redefine the array; we can’t change the sizes of an array! Notice that the value at index 0 is 0. That is the default value for numbers, and since this indice’s purpose is to prop up the array to start at 1, it should stay at 0.

Aside from calling the elements in the array, we can also find the size of the array with arrayName.length. Each array has the length attribute, which stores how many elements are in the array. Be aware that this attribute stores the actual number of elements in the array (which is equal to the number you input in the square brackets when you initialize the array), not the last index in the array. For example, the zero based version of our example would have length store 6, since there are 6 elements in the array, although the index count only goes up to 5. The one based version would have length store 7, since there is one extra filler element. 

We can also have arrays with multiple dimensions (this is the storing arrays in arrays mentioned earlier). To do this, we can just add more pairs of square brackets in the declaration and initialization parts when creating an array. Here is an example of declaring a 2D array.

<data type> [][] arrayName = new <same data type> [<number of rows>][<number of columns>];

We can think about arrays with multiple dimensions like “rectangles” in different dimensions. With 1D arrays, there is only 1 dimension, length, thus the “rectangles” is a line. With 2D arrays, there are 2 dimensions, meaning each index in the length from a 1D array has its own number of indices, making a “rectangle” of indices. With 3D arrays, each index in the 2D array has its won number of indices, making a “rectangular prism” of indices. This can continue to any number of dimensions, but it’s unlikely that we would need anything past the second dimension.
If we created a 2D array with the following syntax, the array would look like the table below: 

int [][] arrayName = new int [3][3];

When we want to access an index in this 2D array, we would specify the row to access first before we specify the column. For example, if we wanted to change the value at row 0 and column 1 to 1, we would type this: arrayName[0][1]=1;. This would turn the array into this:

If we typed arrayName[1][0]=1; instead, the array would turn into this:

Here’s an interesting fact about arrays of multiple dimensions: despite the repeated mentions of them being “rectangles” in this tutorial, each row can actually have a different length! Since arrays of multiple dimensions are just arrays of arrays (of, possibly, more arrays, but we’ll stick to 2 dimensions), each array can have its own array of varying lengths. To change the length of an array, simply type only the row you want altered, as opposed to both the row and the column, and then define it with a different length. For example, if I wanted to redefine the 1st row in the 2D array used in the example above to have a length of 4, I would type this:

arrayName[1] = new int [4];

Notice that everything in row 1 got set back to 0, the default value. That is because the new declaration redefined the array.
