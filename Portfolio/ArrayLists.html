ArrayLists

Everything important about ArrayLists, covered here.

ArrayList Code:

Remember to import the ArrayList library at the top of your program!

import java.util.ArrayList;

Code for creating and manipulating ArrayLists:

ArrayList <Integer> test = new ArrayList<Integer>(); // creating an ArrayList of integers
test.add(0); // adds 0 to the ArrayList
test.add(test.size()); // adds the size of the ArrayList, 1, to the ArrayList
test.add(1, 2); // currently the ArrayList contains 0, 1. After this, it contains 0, 2, 1
test.add(test.get(0)); // adds the value at the first index to the ArrayList (0, 2, 1, 0)
test.set(0, 4); // sets the first value to 4 (4, 2, 1, 0)
test.remove(3); // removes the 4th value (4, 2, 1)

An ArrayList is a class that functions like an array, except it doesn’t have a fixed length! ArrayLists can be as long or short as you like. It works by reserving a chunk of memory for a certain number of elements, and when that number of elements is surpassed, it would double the amount it reserves.

ArrayLists can be summarized as arrays without fixed lengths, but the syntax is quite different. First off, to use them, we must import from the java library:

import java.util.*;

To create an ArrayList, we need to use the following syntax:

ArrayList <data type> arrayName = new ArrayList<data type>();

An important thing to note is that the <> signs actually need to be written here, as opposed to the array syntax, where they shouldn’t be written. Another important thing to note is that, like arrays, ArrayLists are 0 based, meaning the indices start at 0. 

There are a couple important functions with ArrayLists that we should remember. As an example, we will use this ArrayList of Integers, called array. Here is its declaration and initialization, though the adding of all its values is not shown. Note how the type is Integer, not int, meaning that for primitive data types, you must use the wrapper classes as the data type.

ArrayList <Integer> array = new ArrayList<Integer>();

ArrayList.size()

The .size() function returns an integer and allows us to find the size of the ArrayList. It’s like the .length of ArrayLists (it even returns the size as a one-based integer as opposed to a zero-based one, like .length), the difference being that .size() is a function while .length is a variable.

The following call would return 4:

array.size();

ArrayList.get(int index)

The .size() function returns an integer and allows us to find the size of the ArrayList. It’s like the .length of ArrayLists (it even returns the size as a one-based integer as opposed to a zero-based one, like .length), the difference being that .size() is a function while .length is a variable.

The following call would return 17, since the value at index 0 is 17:

array.get(0);

ArrayList.add(Object e) or ArrayList.add(int index, Object e)

The .add() function returns nothing but it allows us to add an element to the ArrayList. Make sure the Object you add is castable to the same type as the Object the ArrayList is supposed to contain. Otherwise, you could get errors! There are two versions of this function: one only takes the Object as a parameter and adds it to the end of the list. The other, overloaded version allows you to specify which index to add the Object to as well as the object. The original Object at that index and everything afterwards gets shifted to the right by one. Once again, be careful with the index, or it could throw an exception!

This call will add an element to the end of the ArrayList:

array.add(15);

This call will add 12 to index 3 and push the values of 6 and 15 to the right:

array.add(3, 12);

ArrayList.set(int index, Object e)

The .set() function returns an object and allows us to replace an Object at a given index with another Object. The original object that would get replaced is returned. Make sure the index is in range!

This call will replace the 6 with a 5:

array.set(4, 5);

ArrayList.remove(int index)

The .remove() function returns an object and removes the object at the index specified. The removed object is returned and all the objects to the right of the removed object has its indices reduced by one. As with every other function with an index parameter, make sure the index is in range!

This call will remove the 4, since it is at index 2 and push the 12, 5, and 15 to the left:

array.remove(2);
